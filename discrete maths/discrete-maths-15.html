Problem 3

In a graph on 101101101 vertices there is an independent set of size 525252. Prove that there is no matching of size 505050 in this graph.

Kinda informal proof through logic

Since there is an independent set of size 525252, then each of these vertices is a part of a potential independent match.

This leaves 101−52=49101-52=49101−52=49 vertices to form those matches with, which simply does not allow one to choose a matching of size 505050.

Formal-esque proof

We know that α(G)\alpha(G)α(G) is the maximum size of an independent set in a graph, τ(G)\tau(G)τ(G) is the minumum size of a vertex cover, and μ(G)\mu(G)μ(G) is the maximum size of a matching, all for graph GGG.

The following identity from the lectures is true:

n−α(G)=τ(G),n-\alpha(G)=\tau(G),n−α(G)=τ(G),

where nnn is the total number of vertices. α(G)\alpha(G)α(G) is obviously at least 525252 per given conditions   ⟹  τ(G)=101−52=49\implies \tau(G)=101-52=49⟹τ(G)=101−52=49.

The following identity is also true:

τ(G)≥μ(G)\tau(G)\geq\mu(G)τ(G)≥μ(G)

Thus,

μ(G)≤49,\mu(G)\leq49,μ(G)≤49,

which means that the maximum size of a matching is 494949 and physically cannot be 505050, q. e. d.

Problem 4

In a graph on nnn vertices there is a vertex cover of size 101010. Prove that in such a graph there is no simple path of length 212121. (In a simple path all vertices are different, and the length of the path is the number of edges in it.)

Since there is a vertex cover of size 101010, then there is an independent set of at least size n−10n-10n−10. Thus, we require at least 21−10=1121-10=1121−10=11 vertices in the independent set. Is this possible?

For a simple path to exist, we need an edge between each two consecutive vertices along the path. Two vertices that are not a part of the vertex cover cannot be consecutive in the path, since there can't be an edge between them (see figure below for a visualization).

Therefore, the longest possible simple path through the graph would look similar to the graph in the figure below (there can be any number of edges between any two black nodes [vertex cover] or between any of the black-white pairings but not between white-white nodes, since that goes against the given conditions).

Per the pigeonhole principle, let's build a best-case scenario graph, placing a vertex between each two vertices that are a part of a cover and starting and ending our path with vertices from the independent set. For the simple path of length 212121 to exist, we need at least 222222 vertices in the graph, which is simply impossible, because there are no two adjacent black vertices we can place a white vertice between, nor can we add any vertices to the beginning or to the end of the graph, thus only resulting in a simple path of length 202020 at the very max, q. e. d.
